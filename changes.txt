Build a table view component with the following behavior:
1. Column Sorting

Every column header must be clickable to sort the table.
Sorting logic must respect each column's data type:

Text columns → alphabetical (A-Z / Z-A)
Number/currency columns → numeric ascending/descending
Date/datetime columns → chronological order
Boolean columns → true/false grouping


Clicking a header should toggle between ascending, descending (and optionally, no sort/default order).
Show a visual indicator (arrow/icon) on the currently sorted column indicating sort direction.

2. Live Search

As the user types in the search box, results should filter in real time (no need to press Enter or a search button).
The search should match against relevant columns (specify which ones, or "all visible columns").

3. Search Results & Pagination Behavior

When a search/filter is active, all matching results must be shown together, without being split across multiple pages — i.e., pagination should not fragment a single filtered result set across pages 1, 2, 3, etc.
Pagination should only apply to the full, unfiltered dataset (or optionally, matched results can still be paginated internally, but the key requirement is: the user should not have to click "next page" to see more of the same search result — either show all matches on one page, or load more within the same context).
If the matched results are large, consider "show all matches" or infinite scroll instead of numbered pagination, so the user isn't forced to navigate across pages to view complete search results.
